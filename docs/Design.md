# 系统设计

想法： 我认为首先系统不应该以拍照读取汉字然后接入接入学习为主要的目标。 
    人要学习必定是从见到到复杂， 这就是为什么我在做生成文章的时候感觉很难的原因，我事先没有想好如何去制作一个整体。仅仅是通过一个图片生成文字的功能去做一个系统， 势必会带来很多问题。

## 基础的学习框架

- 每日结构（固定模板）

    * 1–2小时结构
    * 发音/听力（15分钟）
    * 新句型 + 词汇（30分钟）
    * 跟读 + 复述（30分钟）
    * 应用（说出来）（15–30分钟）

- 按照十五天进行

    * 第一周 发音 + 生存中文 ： 掌握拼音 + 声调， 会最基础表达

        - 学习： 声母， 韵母， 四声 （去 看一下Hello Chinese 怎么做的）
        - 学习简单句子：
            我是 Gordon；
            我是美国人；
            我会说一点中文；
            你好吗？；
            我很好；


    * 拼音学习
        声母 韵母 声调
        随机一个文字给出拼音， 让用户读是什么发音， 给他们回馈
        渐渐单的开始认识拼音

        所有声母列表， 一天学习几个
        所有韵母列表， 一天学习几个
        组合使用一天学习几个
        声调配合声母，韵母出几个字来学习

        用户通过声音来选声母，韵母，声调

        逐渐增加声母， 韵母数量， 进行随机挑选来联系， 通过整个过程来熟悉所有声母韵母组合的学习。

        给一个用户学过的声母， 韵母， 的列表， 可以让用户点亮没有学过的， 逐步学习所有的声母韵母。 完成achievement

        然后可以让用户逐步通过拼音来念词组， 句子， 文章等等。
